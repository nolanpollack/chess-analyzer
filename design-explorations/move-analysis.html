Decisions locked in

  • Tag style: underlined link, dot-separated
  • Best-move-without-description: Variant A (move + eyebrow, no description text yet — will revisit when we have generated descriptions)
  • Eval display: header shows −1.2 → +1.2 on the left of the delta pill, only on player moves; opponent header shows delta only
  • Best-move section appears only when classification is good or worse (delta from best ≥ ~25cp). Excellent/best player moves don't show a best-move panel.
  • Alternatives: up to 3 total (best + 2 alts) when within ~30cp of best
  • Description always rendered (longer for player move, shorter for alts) — no "Why this isn't the move" eyebrow, no "Why your move failed" toggle
  • Tags on alternatives: yes, smaller and quieter than the player-move tags
  • Layout: card has fixed total height; analysis section grows freely (no scroll); moves list flexes to fill the rest with min-height: 280px and is scrollable
Layout tradeoff. No-scroll analysis + non-shifting moves + no awkward whitespace are mutually exclusive. The chosen tradeoff: card is fixed height, analysis grows naturally without scroll, moves list flexes underneath with a min-height. The bottom edge of the moves list is anchored to the card; the top edge moves a bit when analysis grows or shrinks, but there's never empty space above it. In the rare case where analysis content is huge and moves would shrink below 280px, the analysis pushes the card slightly taller — we accept this. If you want the moves list strictly fixed-height instead (card grows above), we can flip it.

Alternative-row tag handling

Alts can carry several tags. Showing them all inline (Variant A) is informative but heavy. Variants B and C hide tags by default and reveal them on click — Variant B uses a chevron, Variant C a count chip. Click a row to expand. Recommendation: B — chevron is a familiar "more here" affordance and gives us a natural home for "view best line" / calculations later.
